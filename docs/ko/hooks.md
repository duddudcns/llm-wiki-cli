# 플러그인에 내장된 안전장치

[English](../en/hooks.md) · **한국어** · [日本語](../ja/hooks.md) · [简体中文](../zh-Hans/hooks.md) · [Español](../es/hooks.md) · [Français](../fr/hooks.md)

Claude Code 플러그인을 설치하면(설치 방법은 [installation.md](installation.md)
참고) 자동 안전장치 세 가지가 켜집니다. 셋 다 `llmw`를 쓰는 데 꼭
필요한 건 아닙니다. 이미 스스로를 보호하고 있는 도구에 얹힌 추가
편의 기능일 뿐입니다.

## 기능 1: AI가 메모를 잘못된 방식으로 고치지 못하게 막기

사실 AI 어시스턴트가 이 도구를 무시하고 다른 파일 고칠 때처럼 위키
메모를 그냥 직접 고치는 걸 기술적으로 막을 방법은 없습니다. 이렇게
되면 자동 백업도, "왜 이걸 바꿨는지" 남기는 필수 메모도, 메모가
제대로 작성됐는지 확인하는 절차도 다 건너뛰게 됩니다. 그리고
실제로도, 다른 지시사항이 우선순위를 차지하고 이 도구를 언급하지
않을 때마다 이런 일이 자주 벌어집니다.

Claude Code 플러그인으로 설치하면, 안전 점검이 이런 상황을
잡아냅니다. AI가 평소 쓰던 파일 편집 도구로 위키 메모를 직접 고치려고
하면, 그 편집은 막히거나(원한다면 먼저 확인을 물어보게 할 수도
있습니다), 대신 어떤 `llmw` 명령어를 써야 하는지 정확히 안내받습니다.
그래서 막힌 채로 있는 게 아니라 바로 올바른 방법으로 다시 시도할 수
있습니다.

이 점검은 이 도구를 쓰는 프로젝트의 위키 메모를 겨냥한 편집만
확인합니다. 그 외 다른 모든 것, 파일을 그냥 읽기만 하는 것도
포함해서, 전혀 건드리지 않습니다.

프로젝트마다 `.llmw/config.toml`에서 이 기능을 끄거나 얼마나 엄격하게
할지 바꿀 수 있습니다:

```toml
[hooks]
wiki_guard = "deny"  # 기본값: 편집을 막고, 올바른 방법을 알려줌
# wiki_guard = "ask"   # 막는 대신 먼저 확인을 물어봄
# wiki_guard = "off"   # 이 프로젝트에서는 이 점검을 끔
```

Windows에서는 이 점검이 동작하려면 "Git Bash"가 설치돼 있어야 합니다.
없다면 이 점검만 그냥 실행되지 않을 뿐, `llmw`에 원래 내장된 안전
규칙들(모든 변경에는 이유가 필요하고, 편집 전에는 백업하는 등)은
그대로 적용됩니다.

## 기능 2: 요청마다 위키를 먼저 확인하도록 AI에게 알려주기

과거의 결정과 실수를 담은 위키도, AI가 새 작업을 시작하기 전에
실제로 들여다봐야 쓸모가 있습니다. 그런데 AI에게 알아서 판단하도록
맡겨두면, 늘 챙겨보는 건 아닙니다. (이건 노트 앱에 위키를 손수
정리해 둔 사람들이 흔히 겪는 문제와 같습니다. 열심히 기록해 두어도,
아무도 봐야 한다고 알려주지 않으면 AI는 결국 이미 적어둔 실수를
그대로 반복하게 됩니다.)

세션을 시작할 때마다 짧은 안내도 하나 보여줍니다. 위키가 이미 있으면
"이 프로젝트에는 메모 N개짜리 위키가 있습니다"라고, 아직 없으면 한
줄로 "`llmw init`을 실행해 보세요"라고 알려줍니다. 그래서 완전히
새로운 프로젝트에서도 AI가 첫 메시지부터 이 도구의 존재를 알게 됩니다.

이를 더 돕기 위해, 보내는 메시지 대부분에도 짧은 안내가 하나씩
따라붙습니다. "이 프로젝트에는 위키가 있으니 먼저 검색해 보라"는
내용입니다. 이건 일부러 단순하게 만든 것입니다. 키워드를 맞춰보면서
방금 보낸 메시지가 위키의 내용과 실제로 관련 있는지 추측하려 들지
않습니다. 그런 자동 추측은 메시지와 표현만 다를 뿐인 메모를 놓치기
쉽기 때문입니다. 대신 매번 그냥 AI에게 확인해 보라고 요청할 뿐이고,
실제 판단(그리고 실제 검색)은 AI 스스로에게 맡깁니다. ("ok"나
"고마워요"처럼 아주 짧은 메시지에는 이 안내가 붙지 않습니다. 새로
시작되는 작업이 없어서 위키와 비교해 확인할 것도 없기 때문입니다.)

이 점검은 위키를 읽어보기만 할 뿐입니다. 요청을 막거나 느리게
만들지 않고, AI가 진행하는 것을 막지도 않습니다. 이건 게이트가
아니라 가벼운 알림입니다.

## 기능 3: 커맨드라인 도구 자체를 최신 상태로 유지하기

이 플러그인에는 작은 실행 프로그램이 들어있지만, 실제 작업은 컴퓨터에
따로 설치된 `llmw`가 처리합니다. 마켓플레이스를 통해 플러그인을
업데이트해도 그 별도로 설치된 프로그램까지 자동으로 업데이트되지는
**않습니다**. 그냥 두면, 본인도 모르는 사이에 오래된 버전을 계속 쓰게
될 수 있습니다.

이걸 막기 위해, 세션을 시작할 때마다 빠른 점검이 하나 돕니다. 컴퓨터에
설치된 `llmw` 버전과 플러그인이 기대하는 버전을 비교하는 거죠. 둘이
다르면(`llmw`가 아예 설치돼 있지 않은 경우도 포함해서), 자동으로 알맞은
버전을 다시 설치해줍니다. 그래서 플러그인을 업데이트하면 커맨드라인
도구도 함께 맞춰지고, 따로 손댈 일이 없습니다.

이미 버전이 맞을 때는 이 점검이 그냥 인터넷 연결도 필요 없는 빠른
로컬 확인일 뿐입니다. 실제로 다시 설치하는 일은 뭔가 어긋났을 때만
가끔 일어납니다.
