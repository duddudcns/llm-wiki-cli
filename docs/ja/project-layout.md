# wikiの整理の仕方と、既存wikiの使い方

[English](../en/project-layout.md) · [한국어](../ko/project-layout.md) · **日本語** · [简体中文](../zh-Hans/project-layout.md) · [Español](../es/project-layout.md) · [Français](../fr/project-layout.md)

## フォルダの整理方法は2通り

デフォルトでは、`llmw init` は `raw/`、`wiki/`、そして裏側のフォルダをプロジェクトの一番上のフォルダに直接置きます。プロジェクトフォルダをすっきりさせたい場合は、代わりに `ai-wiki/` というサブフォルダにすべてまとめることもできます。

```bash
llmw init --layout ai-wiki
```

```text
ai-wiki/
  raw/ wiki/ .llmw/            # 中身は同じ。ただ1段階下に入っているだけ
.claude/skills/llm-wiki/       # これはどちらのやり方でもプロジェクトの一番上に置かれる
.claude-plugin/plugin.json     # これはどちらのやり方でもプロジェクトの一番上に置かれる
```

どちらのスタイルを使っているかは、どのコマンドも自動的に見分けてくれます。最初に `init` したあとは、もう一度指定する必要はありません。すでにシンプルなやり方でメモを作ってある場合は、何も変える必要はありません。

プロジェクトフォルダの外(スクリプトの中など)から `llmw` を実行する場合は、場所さえ教えてあげれば、どちらのフォルダスタイルを使っているか自動で判断してくれます。

```bash
llmw --root /path/to/project status
LLMW_ROOT=/path/to/project llmw status
```

## すでに自分で作ったwikiを `llmw` で使う

このツールを知る前から、自分で作った一連のメモをすでに持っていて、それを `llmw` に管理してもらいたい場合もあるでしょう。その場合は、普通の `init` の代わりに `--adopt` を使ってください。

```bash
llmw init --adopt                  # または: llmw init --layout ai-wiki --adopt
```

これは裏側の検索インデックスを用意しますが、すでにある自分のメモやフォルダを作ったり上書きしたりすることは**絶対に**ありません。あとで `--force` を付けてもう一度実行しても同じです。設定ファイルも同じように保護されるので、自分で加えた設定が知らないうちにリセットされることもありません。(`--adopt` を付けない普通の `llmw init` は動きが違います。こちらはいくつかの最初のメモやフォルダを実際に作成し、`--force` を付けるとそれらを上書きします。なので、普通のやり方はまったく新しい空っぽのプロジェクトでだけ使ってください。)

## ルールが違う既存のメモに合わせる

すでにあるメモが少し違うルールで整理されている場合(たとえば、重要なファイルの一部が `wiki/` フォルダの外にある、`llmw` が期待するのと違うラベルを使っている、など)、何かを作り直す必要はありません。上記のようにwikiを取り込んだあとで、小さな設定ファイルを少し調整するだけで済みます。

```toml
[paths]
# 通常の wiki/ フォルダの外にある、検索対象に含めたい個別のメモファイル
# — たとえば、一番上の階層に置いてあるインデックスや変更履歴のファイルなど。
extra_root_pages = ["index.md", "log.md", "schema.md"]

[lint]
# どのメモにも冒頭に書いておくべき情報の項目。
# 標準では ["type", "status", "created", "updated"] になっている。
# "updated" は代わりに "last_updated" を使っているメモも受け入れる。
required_frontmatter = ["type", "status", "last_updated"]
```

これをするために、既存のメモを何か変える必要はありません。

## Obsidianと一緒に使う

メモはすべてただのテキストファイルなので、`wiki/` フォルダを人気のメモアプリ [Obsidian](https://obsidian.md/) で直接開くこともできます。同じメモに対して、Obsidianの視覚的な地図表示や「どこからリンクされているか」の表示、検索機能が使えます。しかも、AI側の使い方には何の影響もありません。

メモ同士のリンクの扱い方は、Obsidian自身の動きに合わせて設計されていて、細かいクセのある部分にも対応しています。

- Obsidianのリンクの書き方はすべて理解できます。他のメモへの普通のリンク、表示名を変えたリンク、メモ内の特定の見出しへのリンク、そして他のメモの内容を丸ごと取り込む「埋め込み」もです。
- フォルダのパスを含むリンクは、Obsidianと同じ考え方で解釈されます。つまり、リンクが書かれているメモから見た相対位置ではなく、wiki全体の一番上から見た位置として扱われます。
- メモの冒頭には「関連するメモ」をいくつかの書き方で書いておけますが、そのどちらの書き方も正しく認識されます。
- ファイル名にスペースや特殊な文字が入っているリンク(他のツールから書き出したメモによくあります)も、正しく対応づけられます。
- `wiki/` フォルダの外を指しているリンクは、実際にディスク上に存在するかどうかで判定されます。なので、本当にどこにも存在しない場合にだけ「リンク切れ」として報告されます。

**本物のObsidianとの、いくつかの小さな違い:** このツールは、上で触れた「関連するメモ」のリストのように、Obsidian自身の地図表示には出てこない独自のつながりもいくつか理解します。これはこのツール特有の機能だからです。また、違うフォルダに同じファイル名のメモが2つある場合、リンクがどちらのフォルダを指しているか書かれていないと、このツールもObsidianも、時々間違った方を選んでしまうことがあります。それ以外の部分は、ぴったり一致しています。
