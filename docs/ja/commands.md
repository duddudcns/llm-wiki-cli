# 検索の仕組み

[English](../en/commands.md) · [한국어](../ko/commands.md) · **日本語** · [简体中文](../zh-Hans/commands.md) · [Español](../es/commands.md) · [Français](../fr/commands.md)

コマンドの一覧はREADMEの表を見てください。このページでは、`llmw search` が何を「一致した」と判断するのか、その仕組みだけを詳しく説明します。

## 検索は厳しい順に3段階で試される

質問するときと同じ感覚で、そのまま文章で検索できます。特別なキーワードを考える必要はありません。裏側では最大3つのやり方を順番に試していて、前のやり方で何も見つからなかったときだけ次に進みます。つまり、検索語にぴったり一致するメモが、なんとなく似ているだけのメモに順位で負けてしまうことはありません。

1. **厳格(strict)** — 検索語のすべての単語が、そのメモのどこかに出てこないといけません。
2. **緩やか(relaxed)** — どのメモにも一度も出てこない単語(タイプミスや、単語の少し違う形など)は静かに除外され、残りの単語はやはり一致する必要があります。
3. **ゆるい一致(any)** — すべての単語が「あってもなくてもよい」扱いになり、結果はどれだけよく一致しているかで順位づけされます。

`--json` で出力すると、3段階のうちどれで結果が見つかったかも教えてくれます: `{"mode": "strict"|"relaxed"|"any", "dropped_tokens": [...], "results": [...]}`。一番厳格な一致だけがほしい場合は、`--strict` を付けてください。

韓国語のテキストについて: 検索語が、ちょっとした文法的な語尾がついた単一の韓国語の名詞(たとえば `스탯창을` や `포탈에서` のような形)だった場合、その語尾が取り除かれてから一致チェックされます(`스탯창`、`포탈` のように)。そうすることで、素の名詞だけを使っているメモも見つけられるようになります。これは、よくある語尾のうち小さな決まったリストにしか対応していません。すべての単語のすべての活用形をカバーしているわけではありません。ただし、この仕組みのせいでメモが見つからなくなることは絶対にないように作られていて、まれに順位が思ったのと少し違う、という程度の影響にとどまります。
