# プラグインに組み込まれた安全装置

[English](../en/hooks.md) · [한국어](../ko/hooks.md) · **日本語** · [简体中文](../zh-Hans/hooks.md) · [Español](../es/hooks.md) · [Français](../fr/hooks.md)

Claude Codeのプラグイン([installation.md](installation.md) 参照)をインストールすると、2つの自動安全機能がオンになります。どちらも `llmw` を使うために必須というわけではありません。もともとそれ自体で自分の身を守れるツールに、さらに便利さを上乗せしているだけです。

## 機能1: AIが間違ったやり方でメモを編集するのを防ぐ

技術的には、AIアシスタントがこのツールを無視して、他の普通のファイルと同じようにwikiのメモを直接編集してしまうことを止めるものは何もありません。もしそうなってしまうと、自動バックアップも、「なぜこの変更をしたのか」を記録する必須のメモも、メモが正しく書けているかのチェックも、すべて失われてしまいます。そして実際のところ、何か他の指示が優先されていて、このツールのことがまったく触れられていない場面では、これは実際に起こりがちです。

Claude Codeのプラグインとしてインストールしていると、この状況を安全チェックが捕まえてくれます。AIが普段のファイル編集ツールを使ってwikiのメモを直接編集しようとすると、その編集はブロックされます(設定によっては、先に確認を求めるだけにもできます)。そして、代わりにどの `llmw` コマンドを使えばよいかがはっきり示されるので、AIは行き詰まることなく、すぐに正しいやり方でもう一度試せます。

このチェックが見ているのは、このツールを使っているプロジェクトのwikiのメモを狙った編集だけです。それ以外のもの(ファイルをただ読むことも含めて)には一切手を出しません。

このチェックはプロジェクトごとに、`.llmw/config.toml` でオフにしたり、厳しさを変えたりできます。

```toml
[hooks]
wiki_guard = "deny"  # デフォルト: 編集をブロックし、正しいやり方を説明する
# wiki_guard = "ask"   # ブロックする代わりに確認を求める
# wiki_guard = "off"   # このプロジェクトではこのチェックをオフにする
```

Windowsでは、このチェックが動くには「Git Bash」がインストールされている必要があります。入っていない場合は、このチェックだけが動かなくなります。それでも `llmw` 自体に組み込まれた安全ルール(変更には必ず理由が必要、編集前に必ずバックアップを取る、など)は変わらず働き続けます。

さらに、セッションの最初には毎回、ちょっとしたお知らせも表示されます。すでにwikiがある場合は「このプロジェクトにはN個のメモがあるwikiがあります」、まだない場合は「`llmw init` を実行するとよいです」という一行のヒントです。これにより、まったく新しいプロジェクトであっても、AIは最初のメッセージからこのツールの存在を知ることができます。

## 機能2: コマンドラインツール自体を自動的に最新の状態に保つ

このプラグインには小さな補助プログラムが含まれていますが、実際の作業をしているのはパソコンにインストールされている、別のコピーの `llmw` です。マーケットプレイスからプラグインを更新しても、その別のコピーは自動的には更新され**ません**。放っておくと、気づかないうちに古いバージョンを使い続けてしまうことになりかねません。

これを防ぐため、セッションの最初に毎回ちょっとしたチェックが走ります。パソコンにインストールされている `llmw` のバージョンと、プラグインが期待しているバージョンを見比べるのです。もし食い違っていたら(そもそも `llmw` がまだインストールされていない場合も含めて)、正しいバージョンを自動的に入れ直してくれます。つまり、プラグインを更新するだけで、コマンドラインツールの方も自動的につじつまが合うようになっていて、あなたが何かする必要はありません。

バージョンがすでに一致している場合、このチェックはインターネット接続も不要な、素早いその場限りの確認で終わります。実際に入れ直す処理が走るのは、何かが食い違っているまれなときだけです。
