# フック：エージェントの正直性を保ち、CLI を同期状態に保つ

[English](../en/hooks.md) · [한국어](../ko/hooks.md) · **日本語** · [简体中文](../zh-Hans/hooks.md) · [Español](../es/hooks.md) · [Français](../fr/hooks.md)

Claude Code プラグイン（[installation.md](installation.md) を参照）は 2 つのフックをインストールします。どちらも `llmw` を使用するために必須ではありません — それらはフックが実行されたかどうかに関わらず独自のセーフティルールを既に実装している CLI の上に層を置いた利便性です。

## PreToolUse：wiki ガード

エージェントが Claude Code スキルを無視し、`llmw` の代わりに独自のファイル編集ツールを使用して `wiki/*.md` または `raw/**` を直接編集するのを何も止めません — これは `--reason` 監査ログ、フロントマター検証、自動バックアップをスキップしており、異なる相互に競合する説明が有効であり `llmw` を言及しないことがあるたびに実践的に発生します。

Claude Code プラグイン（ bare `llmw init` プロジェクトスキル ではない）としてインストールされると、`PreToolUse` フックはハーネスレベルでそのギャップを閉じます：`wiki/*.md` または `raw/**` をターゲットとするネイティブ `Edit`/`Write`/`NotebookEdit` 呼び出しは拒否されます（または、設定に従い、確認プロンプトに変わります）、拒否メッセージは実行すべき正確な `llmw` コマンドを名付けます — したがってエージェントの次のアクション は行き止まりではなく、ワンライナーの書き直しです。

ガードは編集/書き込み/NotebookEdit 呼び出しだけを見ます。そのターゲットがファイルから上へ歩くことで解決する（`llmw` が独自のプロジェクトルートを見つけるのと同じ方法）実際の llmw プロジェクトの `wiki/*.md` または `raw/**` — その他のすべて、素の `Read` を含む、変わらずに通過し、`Bash` コマンドを検査しません（シェル文字列ポリシングはそれ自身のフォルスポジティブの沼地であるため、`wiki/log.md` の監査証跡と `llmw lint` は代わりにそのギャップの検出層として残ります）。

プロジェクトごとに `.llmw/config.toml` で設定または無効化：

```toml
[hooks]
wiki_guard = "deny"  # デフォルト：ブロック、llmw 修正を名付けるメッセージ付き
# wiki_guard = "ask"   # ブロックの代わりに確認をプロンプト
# wiki_guard = "off"   # このプロジェクトのガードを無効化
```

両方のフックは Windows 上で Git Bash を必要とします（Claude Code は Git Bash がインストールされていない場合 PowerShell に戻り、これらのシェル形式フックはサポートされません） — 他の場所では、`llmw` 自身のセーフティゲート（理由必須、パス制限、フロントマター検証、書き込み前バックアップ）はフックが実行されたかどうかに関わらず保持されます。

また、毎回セッション開始で短い `SessionStart` メモをコンテキストにドロップします：`.llmw` が既に存在するときは「このプロジェクトは llmw wiki を持っている」（ページ数付き）、またはそれがまだ初期化されていないときは ワンライナー「`llmw init` を実行」ヒント — したがって プロジェクト CLAUDE.md なしの空白環境、そして wiki がすべて初期化されていない でさえ、ターン 1 で `llmw` を発見します。

## SessionStart：自己修復 CLI インストール

`plugin/bin/llmw` はバンドルされた Python 配布ではなく、薄いディスパッチャです — PATH の実際の `llmw` に出ます。マーケットプレイスからプラグインを更新することはプラグイン自身のファイル（スキル、フック）だけを更新します。それはスタンドアロンバイナリに触れ**ません**。そのままにすると、プラグイン更新のインストールはそれをサイレント同期させて古い CLI で実行したままになることを意味します。

`SessionStart` フック（`plugin/hooks/session-start.sh`、`plugin/hooks/hooks.json` を使用して配線）はそのギャップを閉じます：毎回セッション、インストールされた `llmw --version` をこのプラグインバンドルが宣言する（`plugin/.claude-plugin/plugin.json`）バージョンと比較します。ミスマッチの場合 — 「すべてインストールされていない」を含む — `uv tool install --force` でインストール（`pip install --user --force-reinstall` へのフォールバック）、一致する `git` タグ にピン (`git+...@v<version>`)、プラグインマーケットプレイス更新も別の手動 `uv tool upgrade llmw` なしでスタンドアロン CLI バイナリを同期します。

バージョンが既に一致する場合、チェックはただ 1 つのローカル `llmw --version` 呼び出し（ネットワーク不要）毎回セッション — インストール再パスは正確なバージョンドリフト上でのみ実行され、大体リリースごとに 1 回。
